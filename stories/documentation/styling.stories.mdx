# Styling

One of the key features of the `react-language-switcher` package is its flexibility when it comes to styling the `<LanguageSwitcher />` component.

It is possible to override the `className` prop for each element, or override the whole component for each element separately, as well as mix and match these combinations.

## Default styling

By default, the component is completely unstyled, as we don't want to enforce our opinions on you.

It will render a list in the following html format:

```html
<ul>
  <li>
    <button>default language</button>
  </li>
  <li>
    <button>...additional languages</button>
  </li>
</ul>
```

## Applying custom styling

For the purpose of this example, we will use vanilla css, but you can find examples for different approaches inside the Examples section.

You can style the component in 3 ways.

### Overriding classes

There are 5 separate class properties that you can override, visible in the example below.

```tsx
  <LanguageSwitcher
    containerClassName="language-switcher"
    listClassName="language-switcher__list"
    itemClassName="language-switcher__item"
    buttonClassName="language-switcher__button"
    buttonActiveClassName="language-switcher__button--active"
/>
```

This approach will keep the default component elements, and apply your classes to produce a result like this:

```html
<div class="language-switcher">
  <ul class="language-switcher__list">
    <li class="language-switcher__item">
      <button class="language-switcher__button language-switcher__button--active">
        default language
      </button>
    </li>
    <li class="language-switcher__item">
      <button class="language-switcher__button">
        ...additional languages
      </button>
    </li>
  </ul>
</div>
```

### Overriding components

There are 4 separate component properties that you can override.

```tsx
  <LanguageProvider>
    <LanguageSwitcher
      containerClassName="language-switcher"
      containerComponent={Container}
      listComponent={List}
      itemComponent={Item}
      buttonComponent={Button}
    />
</LanguageProvider>
```

This approach will keep the default component elements, and apply your classes to produce a result like this:

```html
<div class="language-switcher">
  <ul class="language-switcher__list">
    <li class="language-switcher__item">
      <button class="language-switcher__button language-switcher__button--active">
        default language
      </button>
    </li>
    <li class="language-switcher__item">
      <button class="language-switcher__button">
        ...additional languages
      </button>
    </li>
  </ul>
</div>
```

# Support

We added support for styling via:

- vanilla-css
- css-modules
- styled-components
- other CSS-in-JS frameworks such as emotion or aphrodite

If your solution of choice does not work, let us know and we will gladly take into consideration if we can somehow help you out.
